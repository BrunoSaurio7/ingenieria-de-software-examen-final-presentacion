# Estándar Q-Holístico

Arquitectura, Talento y Blindaje de Datos > Una metodología de calidad para el software sostenible.

Este repositorio aloja la presentación interactiva del Estándar Q-Holístico, un marco de trabajo diseñado para redefinir la calidad del software moderno. A diferencia de los estándares tradicionales, este enfoque integra la excelencia técnica con la sostenibilidad humana y la seguridad de los datos.

# Los 3 Pilares del Estándar

El estándar evalúa el software sobre una base de 100 puntos, divididos en:

1. Arquitectura Ágil y Escalable (35%): Capacidad del sistema para adaptarse y crecer (Microservicios, Auto-escalado, CI/CD).
2. Bienestar y Sostenibilidad del Equipo (30%): Enfoque en la salud mental y física de los desarrolladores ("Cero Héroes", Salud del On-Call).
  - Regla de Veto: Si el puntaje en este pilar es < 50%, el software NO se certifica.
3. Seguridad: Confidencialidad e Integridad (35%): Protección robusta de datos (Cifrado E2E, Logs inmutables).

# Cómo ver la presentación

## Opción 1: GitHub Pages (Recomendado)

Visita la URL proporcionada en la sección "Environments" o "Deployments" del repositorio.

## Opción 2: Ejecución Local

1. Clona este repositorio:
git clone [https://github.com/tu-usuario/estandar-q-holistico.git](https://github.com/tu-usuario/estandar-q-holistico.git)
2. Navega a la carpeta del proyecto.
3. Abre el archivo index.html en cualquier navegador web moderno (Chrome, Firefox, Edge, Safari).
No se requiere instalación de dependencias ni servidores complejos, ya que utiliza librerías vía CDN.
